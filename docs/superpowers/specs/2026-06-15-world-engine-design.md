# 世界引擎 — 文字冒险游戏 设计文档

> 日期: 2026-06-15 | 状态: 已确认

## 概述

一个「世界引擎」框架。创作者用自然语言定义世界卡，玩家选择世界卡进入冒险。AI 驱动 NPC 反应、剧情推进和世界状态。前端网页渲染流式对话、六边形地图、物品栏和状态面板。

## 角色

| 角色 | 做什么 |
|------|--------|
| 玩家 | 选世界卡 → 冒险 |
| 创作者 | 自然语言描述世界 → 生成/修改世界卡 |
| 世界引擎 | 读世界卡 → 驱动 AI → 渲染世界 |

## 技术栈

- **前端**: Next.js (React/TypeScript) + Tailwind CSS
- **后端**: Next.js API Routes
- **AI**: Claude API（世界卡作者可配 Key）
- **渲染**: 流式 Server-Sent Events

## 世界卡结构

创作者用自然语言描述，框架 + AI 辅助整理为结构化世界卡：

| 字段 | 内容 | 示例 |
|------|------|------|
| 世界观 | 背景、规则、基调 | 蒸汽朋克，魔法与机械并存 |
| 种族/势力 | 可选阵营 | 天空贵族、地底工匠 |
| 初始场景 | 玩家起点 | 坠毁飞艇残骸中醒来 |
| 核心冲突 | 驱动问题 | 为什么天空城在坠落？ |
| 视觉风格 | 主题色、氛围 | 铜色+深蓝、齿轮纹理 |

世界卡来源：预设 / AI 生成 / 玩家手动修改。

## 核心游戏循环

```
玩家输入 → 组装上下文 → AI 编排(一次调用) → 结构化JSON → 前端消费
```

- 一次 API 调用完成所有计算
- 后端缓冲流式帧，到齐后前端同步呈现（打字机动画 + 选项 + 状态更新）
- 选项由 AI 根据上下文动态生成

## 存档系统

完整世界快照：玩家属性 + 物品 + 地图探索状态 + 世界标记 + NPC 记忆 + 任务状态

- 3 手动档 + 1 自动档（移动到新地点自动存）
- 存储：localStorage（快速）+ 可选云端同步

## MVP 范围

第一版只包含：
1. 预设世界卡选择 → 进入游戏
2. 流式对话 + 选项推进（打字机效果）
3. 轻量数值系统（属性条 + 选项条件 + 效果）
4. 存档/读档（localStorage）

排除（后续版本）：
- 六边形地图
- 物品栏
- NPC 记忆系统
- 创作者工坊
- AI 生成世界卡
- 云端同步
